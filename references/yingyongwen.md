# 应用文写作 - 批改标准与答题框架

> 此文件由 SKILL.md 在识别为「应用文写作」后加载。

## 答题技巧总结

### 核心概念区分（必须牢记）

- **对策定义**：材料中人物/机构**已经实施或正在实施**的具体行动措施，以行动、动作为主体，效果为补充的语段。对策是材料中的既定事实，用户只需转述提取，不可自行编造。
  - 识别标志：材料中出现"建立了""实施了""开展了""推行了"等过去时或进行时的行动描述
  - 权重地位：**绝对主导**，占总分值70-80%权重

- **建议定义**：用户根据材料中发现的问题，**自行提出**的解决方案或改进措施。建议是用户的主观判断，需要基于问题分析得出。
  - 权重地位：**次要补充**，占总分值5-10%权重，篇幅严格控制

### 对策权重铁律（绝对核心）

- 应用文以材料中的对策为绝对主体，对策类要点占总分值70-80%权重
- 问题类内容仅作背景补充，占总分值10-15%权重，必须高度精简
- 建议类内容占总分值5-10%权重，仅在材料缺乏对策时适度补充
- 权重分配顺序：材料对策要点 >> 问题分析 > 现状描述(背景) > 用户建议
- 篇幅分配铁律：对策类内容占正文篇幅70-80%，问题类内容≤3行，建议类内容≤2行
- 找点优先级：优先提取材料对策要点，问题类要点仅作补充，建议要点最后考虑

### 四句顺口溜（应用文解题总纲）

1. **前提算点手段，点其逻辑思维**：估算题目大概要答几句话，确定段数匹配，保留原文逻辑
2. **对策写成大条，答题务必分段**：以议论文为主体，讲究3W原则，对策为主，合并提炼，前置提炼
3. **每句写的通顺，小词儿随意串联**：保证语言流畅，灵活串联，避免生硬
4. **格式语言要求，看清题目再干**：掌握格式要素，根据文体调整语言风格

### 五把钥匙（分段方法）

1. **讲逻辑**：3W原则（是什么、为什么、怎么办）
2. **讲顺序**：时间轴分段（前期准备、中期施工、后期试运行）
3. **讲过渡**：找材料里的过渡段或过渡句
4. **讲故事**：三段式（背景、经过、结果）
5. **讲规矩**：兜底方法（题目规矩、材料规矩）

## 字数-答题条数对应表

```
⚠️ 应用文对策条数铁律（绝对不可违反）：
字数限制 | 最大条数 | 材料要点数 | 合并原则
--------|----------|----------|----------
300字以下 | 最多4条 | 4-6个 | 大多数点独立成条，少部分2点合1条
350字   | 最多5条 | 4-5个 | 大多数点独立成条，少部分2点合1条
400字   | 最多6条 | 5-6个 | 大多数点独立成条，少部分2点合1条
450字   | 最多7条 | 5-7个 | 大多数点独立成条，少部分2点合1条
```

**合并策略铁律**：
- 材料对策要点优先独立成条（70-80%独立）
- 问题要点优先合并（为对策让出空间）
- 建议要点最小化处理

## 特殊题型处理

- **对策类应用文**：倡议书、工作汇报、考察报告、经验交流稿、通知通告（必须对策为主）
- **宣传类应用文**：宣传稿、讲话稿、导言（可能重点是why而不是how）
- **汇报类应用文**：情况说明、调研报告、工作总结（可能what是重点）

## 具体解题步骤

1. **审题定位**：文体识别、要求分析、字数估算、对象确定
2. **材料分析**：段数统计、要点提取、逻辑梳理、重点识别
3. **结构设计**：分段方案、逻辑安排、字数分配、层次规划
4. **内容组织**：前置提炼、要点合并、语言加工、逻辑检查

## 要点提取技巧

### 对策识别与提取
- 材料尾段明确的对策是真对策，必须直接提取，不可修改
- 材料中间的对策仅作参考，需结合上下文判断可行性
- 禁止反推建议：如"资金不足"不能自行写"加大资金投入"

### 人物话语要点提取
- 首句总领话语：通常概括整段核心内容
- 尾句引领话语：通常引出后续内容
- 中间阐述话语：通常包含具体措施或问题描述
- 提取原则：将人物话语转换为客观表述（如"我们设立了XX部门"→"设立XX部门"）

### 自然段找点关键技巧
- **转折词**："但是""然而""不过"等，转折后往往是重点
- **序数词**："第一""首先""其次"等，标志要点逻辑顺序
- **最高级词**："最""极""非常"等，突出重要性的关键信息
- **指示词**："这""那""此""该"等，通常指向前文重要内容

### 段落处理逻辑

| 段落类型 | 处理方式 |
|---------|---------|
| 独立段落 | 包含完整对策或问题，优先提取 |
| 补充段落 | 提供细节或例证，合并到相关主段落 |
| 数据段落 | 仅包含具体数字，直接舍弃 |
| 过渡段落 | 起连接作用，通常舍弃 |
| 描述性段落 | 纯描述性语言，找不到要点时可忽略 |

**舍弃优先级**：数据段落 > 过渡段落 > 描述性段落 > 补充段落 > 问题类独立段落 > 对策类独立段落

## 逻辑组织原则

### 答题顺序铁律

1. **逻辑顺序优先**：核心对策→重点对策→一般对策→问题背景→建议补充
2. **层次递进**：宏观政策→中观措施→微观操作→效果评估
3. **因果逻辑**：问题分析→对策措施→预期效果→保障机制

### 材料逻辑线索识别
- 出现"阶段""前期""中期""后期"→严格按时间顺序分大条
- 涉及"经济""社会""文化""生态"→按重要性顺序分大条
- 涉及"政府""企业""社会"等主体→按主导性顺序分大条

### 常用组织方式
1. 按方面组织：经济建设、社会建设、文化建设、生态建设
2. 按阶段组织：前期准备、规划设计、实施推进、总结完善
3. 按主体组织：政府层面、企业层面、社会层面、个人层面
4. 按层次组织：宏观层面、中观层面、微观层面

## 格式要求

- 倡议书、工作汇报、考察报告、经验交流要写成'大条'
- 穿靴：每'大条'开头加**密集型前置提炼**（如"一、强化政策保障"）
- 戴帽：条目后补充具体案例（但不包括具体数字）
- 层次分明：'大条'下可再分二级条目
- 若要求'生动'、'有号召力'等字样，需在回答结尾处呼吁

## 批改评分细则

### 评分权重分配
- **内容评分（70%）**: 材料对策要点占70-80%权重，问题占10-15%，建议占5-10%
- **结构评分（10%）**: 答题结构完整性、段落过渡自然性
- **语言评分（20%）**: 标注语病、冗余表述

### 打分执行标准

每次批改必须包含：
1. 分项得分明细
2. 扣分原因说明
3. 最终分数计算
4. 等级评定
5. 针对性建议：至少3条
6. **应用文条数检查**：必须严格检查条数是否超过铁律限制
7. **对策权重检查**：对策类要点是否占主体地位（70-80%）
8. **要点完整性检查**：是否遗漏关键材料对策要点

### 要点评分细则（材料对策权重绝对优先）

**材料对策要点评分（占要点总分70-80%）**:
- 完全正确且表述准确：满分
- 要点正确但表述不够准确：扣20-30%
- 要点部分正确：扣40-60%
- 要点错误或遗漏：0分（重大扣分项）
- 每个材料对策要点缺失都是重大扣分

**问题/现状要点评分（占要点总分10-15%）**:
- 缺失影响相对较小，不是主要扣分项

**建议要点评分（占要点总分5-10%）**:
- 建议合理且针对性强：适度加分
- 建议过多或喧宾夺主：扣分（违反权重原则）

### 表述评分细则
- 语言准确性（30%）：术语使用、表达精准度
- 条理性（30%）：逻辑清晰、层次分明
- 顺序合理性（25%）：要点排列顺序是否符合逻辑，对策是否优先
- 过渡衔接性（15%）：段落间、要点间是否有自然过渡

### 格式评分细则
- 字数控制（40%）：超标1-20字扣10%，21-50字扣30%，51字以上扣50%
- 段落安排（30%）
- 结构层次（20%）：各大条层次是否分明
- 标点使用（10%）

## 打分示例模板

```
【题目】请写一份关于XX工作的汇报材料。（25分，不超过400字）

【考生答案】[考生作答内容]

【批改评分】
✅ 要点得分：14/20分（材料对策权重优先评分）
- 材料对策要点（占80%权重）：
  - 要点1：政策保障措施 ✓
  - 要点2：人员配备安排 ✓
  - 要点3：技术支撑体系 ✗（-6分，材料对策要点缺失，重大扣分）
- 问题/现状要点（占15%权重）：
  - 要点4：资金投入情况 ✓
- 建议要点（占5%权重）：
  - 要点5：改进建议 ✓

✅ 应用文条数检查：400字题目最多6条，实际5条，符合铁律 ✓
- 材料对策要点数量：3个，占总要点数75%，符合70-80%要求 ✓
- 材料对策要点篇幅：占正文篇幅78%，符合要求 ✓
- 问题要点篇幅：2行，符合≤3行要求 ✓
- 建议要点篇幅：1行，符合≤2行要求 ✓

✅ 表述得分：18/25分
- 语言准确性：较好（-2分）
- 条理性：有前置提炼，对策逻辑清晰（-2分）
- 顺序合理性：对策优先，问题后置，符合逻辑（-1分）
- 过渡衔接性：段落间缺乏自然过渡（-2分）

✅ 格式得分：15/20分
- 字数控制：395字，符合要求 ✓
- 段落安排：分段合理 ✓
- 结构层次：各大条层次分明 ✓
- 标点使用：基本正确，有1处错误（-5分）

【最终得分】：14×0.7 + 18×0.2 + 15×0.1 = 14.9分
  （要点14×0.7=9.8 + 表述18×0.2=3.6 + 格式15×0.1=1.5）
【等级评定】：不及格（59.6%）

【改进建议】：
1. 补充核心材料对策要点：技术支撑体系、监督考核机制
2. 强化材料对策权重：确保对策类内容占正文70-80%
3. 严格控制建议要点：建议仅作补充，篇幅≤2行
4. 优化合并策略：优先合并问题类和建议类要点为对策让空间
5. 密集型前置提炼：用4-8字精准概括每条要点核心
6. 强化过渡衔接表达
```
